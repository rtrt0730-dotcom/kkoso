# /업데이트 — 자동 배포 명령어

변경된 파일을 자동으로 커밋하고 GitHub에 푸쉬하는 한 번의 명령어로 배포합니다.

## 실행 흐름

### 1️⃣ 변경된 파일 확인
```bash
git status
```
- 수정되거나 생성된 모든 파일 목록 표시
- 추적되지 않은 파일 확인

### 2️⃣ 커밋 메시지 자동 생성
변경 내용을 분석하여 자동으로 커밋 메시지 생성:
- **파일 추가**: `파일명 추가`
- **파일 수정**: `파일명 업데이트`
- **다중 변경**: `여러 파일 업데이트`

예시:
```
업무대시보드.html 업데이트: 모바일 최적화 추가
매출보고서.html 업데이트: 카테고리 이모티콘 변경
```

### 3️⃣ GitHub에 푸쉬
```bash
git add .
git commit -m "자동 생성된 메시지"
git push origin main
```

### 4️⃣ 완료 메시지
```
✅ 배포 완료!
   📝 커밋: [commit hash]
   🌐 URL: https://github.com/rtrt0730-dotcom/kkoso
   🚀 라이브: https://kkoso-n4hw.vercel.app
```

## 사용 예시

```bash
/업데이트
```

## 주의사항

- ⚠️ 민감한 파일 (.env, 비밀번호) 확인 후 푸쉬
- 🔄 로컬 변경사항이 모두 커밋됨
- 🌐 GitHub + Vercel 자동 배포 시작

## 수동 푸쉬 (필요시)

자동화가 안 될 경우 수동 실행:
```bash
git add .
git commit -m "커스텀 메시지"
git push origin main
```
